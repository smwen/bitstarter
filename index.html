Knitting, crocheting, and other yarn crafts have become "cool" again within the past 10 years, with the internet providing easy access to patterns and yarn suppliers. More and more yarn artists are now designing their own patterns, intended for sale or free giveaway. However, software support for pattern design is a small underserved market, and design software is often expensive or not affordable for the average knitter. Additionally, since pattern notation is not well standardized, thus adding difficulties to publishing patterns understandable by the masses.

To allow better support for knitting/crocheting pattern design, we will be writing open source APIs to aid in software development for yarn arts. We will write packages for notating yarn-based project patterns, and for accessing Ravelry, a personal knitting projects website.

FAQ

What are the funds to be used for?
The funds are used to pay for domain name/server costs, and any listing or other costs.
What's the scoop on standardization?
Graphical notation for basic knitting stitches (knit, purl, ssk, k1, etc.) are not standardized; published notations vary widely from dots, slashes, to shading and colors. This API plans to give the user the option to switch between some common notations with low overhead... it beats redrawing diagrams on graph paper by hand!
